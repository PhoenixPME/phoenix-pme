# Asynchronous Communication Protocol

## For Developers Reaching Out
1. Review our [State Machine Specification](./diagrams/auction_state_machine.md)
2. Check if you can contribute to any [open issues](https://github.com/PhoenixPME/coreum-pme/issues)
3. Submit a draft PR or technical comment on relevant issues
4. If serious about collaboration, email technical questions to gjf20842@gmail.com

## For Community Members
1. Read our [MANIFESTO.md](../MANIFESTO.md) and [DAO_OPERATIONS.md](../DAO_OPERATIONS.md)
2. Join discussions via GitHub Issues (no live chat required)
3. Submit feedback via the governance feedback template

## Core Principle
We believe in written, asynchronous communication that:
- Allows deep thinking before responding
- Creates permanent documentation
- Respects different communication styles
- Values technical merit over social skill
